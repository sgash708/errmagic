# errmagic v0 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** ブラウザJSエラーを捕捉し、直近約60秒のrrwebセッションリプレイを添えて任意のエンドポイントへPOSTするnpmパッケージ（+ ローカル再生ビューア）を作る。利用側アプリから git 依存で利用する。

**Architecture:** 依存は rrweb のみ（React連携は peerDependency・subpath export）。録画はメモリ上のリングバッファのみで常時送信はしない。エラー発生時に gzip(base64) してエラー報告と一緒に fire-and-forget 送信。

**Tech Stack:** TypeScript / tsup（ESM + d.ts）/ vitest + jsdom / rrweb ^2.x（record） / rrweb-player（ビューアのみ） / pnpm

## Global Constraints

- **デフォルト全マスク**: 全テキスト・全入力値をマスク、`img,video,canvas` は blockSelector で全ブロック。`.rr-unmask` クラスでのみテキストマスク解除
- **レポーター自身がエラーループを起こさない**: すべての公開経路を try/catch し、送信失敗は握りつぶす。再入ガードを持つ
- **同一エラーはクライアント側で 5 分間デデュープ**（`dedupeWindowMs` デフォルト 300000）
- **リプレイ添付はセッション毎・同一エラーにつき 1 回**
- CompressionStream 非対応ブラウザではリプレイなしでエラーのみ送信（機能検出）
- `dist/` はコミットする（git 依存で prepare ビルド不要にするため）
- コミットは Conventional Commits + 日本語

## 送信ペイロード（サーバー側と共有する契約）

```jsonc
POST {endpoint}  Content-Type: application/json
{
  "app": "my-app",
  "name": "TypeError",                  // エラークラス名
  "message": "Cannot read ...",         // max 2000 chars に切り詰めて送る
  "stack": "TypeError: ...\n at ...",   // max 20000 chars
  "url": "https://app.example.com/...",
  "user_agent": "Mozilla/...",
  "occurred_at": "2026-07-15T00:00:00.000Z",
  "replay": "<base64(gzip(JSON.stringify(rrweb events)))>" ,  // 無い場合は null
  "replay_format": "rrweb-gzip-base64"                        // replay が null なら null
}
```

## 公開API（正確に守る）

```ts
// "errmagic"
export interface ErrmagicOptions {
  endpoint: string;                 // 必須。POST先URL
  app: string;                      // 必須。例 'my-app'
  replay?: boolean;                 // default true
  dedupeWindowMs?: number;          // default 300_000
  beforeSend?: (report: ErrorReport) => ErrorReport | null;  // nullで送信中止
}
export function initErrmagic(options: ErrmagicOptions): void;   // 二重init時は後勝ちで警告なし上書き
export function reportError(error: unknown, context?: Record<string, unknown>): void; // 手動報告

// "errmagic/react"
export class ErrmagicErrorBoundary extends React.Component<
  { fallback?: React.ReactNode; children: React.ReactNode }
> {}  // componentDidCatch で reportError し、fallback（default: 日本語の簡易エラー画面+再読み込みボタン）を表示
```

## タスク分割

### Task 1: パッケージ雛形と dedupe / transport（TDD）
package.json（name errmagic, private: true, type module, exports: "." と "./react", files: dist, peerDependencies react >=18 optional）、tsup（entry: src/index.ts, src/react.tsx, format esm, dts, sourcemap）、vitest（jsdom）。
`dedupe.ts`: `class Deduper { constructor(windowMs); shouldReport(key: string, now?: number): boolean }` — 同一keyはwindow内でfalse。テスト: 初回true / 窓内false / 窓超過true。
`transport.ts`: `buildReport(err, ctx, opts) => ErrorReport`（Error以外の値も安全に文字列化、message/stack切り詰め）、`send(endpoint, report): void`（fetch keepalive, .catch(()=>{})、JSDOMではfetchをvi.stub）。テスト: Error/string/undefinedからの構築、切り詰め、beforeSendがnullなら送らない。

### Task 2: recorder（rrwebリングバッファ）（TDD）
`startRecorder()`: rrweb.record を maskAllInputs: true / maskTextSelector: '*' / （インストールしたrrwebのバージョンに unmaskTextSelector があれば `.rr-unmask` を指定、無ければ maskTextFn で `el?.closest('.rr-unmask')` のとき素通し）/ blockSelector: 'img,video,canvas' / checkoutEveryNms: 30_000 で起動。emit(event, isCheckout) で2セグメント式リングバッファ（isCheckoutで旧セグメント破棄→現行を旧に）。
`takeReplay(): Promise<string | null>`: バッファをJSON化→CompressionStream('gzip')→base64。非対応ならnull。`hasAttachedFor(key)` でセッション毎1回制御。
テスト: rrweb.record をモックしてemitコールバックを直接叩き、チェックアウトでバッファがローテーションすること・takeReplayがbase64を返すこと（DecompressionStreamで往復検証）・CompressionStream未定義でnullを返すこと。

### Task 3: index.ts 統合 + react.tsx（TDD）
`initErrmagic`: window.onerror（既存ハンドラ保存して連鎖）+ unhandledrejection を addEventListener。再入ガード（報告処理中に発生したエラーは無視）。dedupeキーは `name|message先頭200|stack1行目`。replay添付は Deduper とは別に「エラーキー毎に1回」。
`reportError`: init前に呼ばれたら何もしない。
`react.tsx`: ErrorBoundary（componentDidCatch → reportError(error, {componentStack})、fallback表示）。
テスト: window.dispatchEvent(new ErrorEvent…) → fetch stub が呼ばれる / 同一エラー連投で1回 / beforeSend=nullで0回 / ErrorBoundaryのレンダリング（@testing-library/react）。

### Task 4: viewer + ドキュメント + dist コミット + v0.1.0 タグ
`viewer/index.html`: rrweb-player（unpkg CDN）+ pako（CDN）で、(a) .json.gz ファイルの file input、(b) `?src=<presigned URL>` のfetch、の両方から再生。使い方コメントを日本語で先頭に。
README.md: インストール（`pnpm add github:sgash708/errmagic#v0.1.0`）、init例、マスキング方針（.rr-unmask）、ビューア手順（`aws s3 presign s3://your-error-replay-bucket/<key>` → viewerに貼る）。AGENTS.md: リポジトリ概要・ビルド/テストコマンド・distコミット運用。
`pnpm build` して dist/ をコミット。`git tag v0.1.0` + push。
