⚡errmagic — アーキテクチャ全体図

外部SaaSなしのSentryライクなブラウザ用エラーレポーター。紫の枠がこのライブラリの守備範囲で、青の点線枠（受信API・保存先）は利用者側で用意する。 インストール: pnpm add github:sgash708/errmagic#v0.1.0（dist コミット済み・prepare ビルド不要）

errmagic 本体
依存は rrweb のみ。録画はメモリ上リングバッファのみ（常時送信なし）
  • 録画: rrweb リングバッファで直近約60秒を保持（30秒毎checkout・2セグメント式）
  • マスキング: 全テキスト・全入力値をデフォルトマスク、img,video,canvas は全ブロック。.rr-unmask のみ解除可
  • デデュープ: 同一エラーは5分間再送信しない／リプレイ添付はセッション毎・同一エラー1回
  • 送信: エラー発生時に gzip + base64 して fetch keepalive で fire-and-forget
※ 送信失敗・レポーター内部の例外はすべて握りつぶし、アプリを壊さない（再入ガードあり）

📦 送信ペイロード
（API と共有する契約）
POST {endpoint} Content-Type: application/json { "app": "my-app", "name": "TypeError", "message": "...", // message は最大2000文字 "stack": "...", "url": "https://...", "user_agent": "...", // stack は最大20000文字 "occurred_at": "2026-07-15T00:00:00.000Z", "replay": "<base64(gzip(rrweb events JSON))>", "replay_format": "rrweb-gzip-base64" } // 無ければ両方 null
replay は「base64 → gzip バイナリ」の二重包み。API側は base64 デコードだけして gzip のまま保存すれば、ビューアがそのまま読める。
⚡ エラーループ防止: 公開経路（init / reportError / リスナー / ErrorBoundary）はすべて try/catch。送信失敗は握りつぶし、報告処理中のエラーは再入ガードで無視。
🔒 プライバシー: デフォルト全マスク（テキスト・入力値）。.rr-unmask を付けた要素のテキストだけ解除。入力値のマスクは解除不可。
🧭 フォールバック: CompressionStream 非対応ブラウザではリプレイなしでエラーのみ送信（機能検出）。
🏷 バージョン運用: git 依存（github:sgash708/errmagic#vX.Y.Z）。dist/ コミット済みのため consumer 側でのビルド不要。
